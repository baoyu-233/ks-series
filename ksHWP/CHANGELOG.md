# ksHWP 修改日志

## 2026-07-23

- 修复服务重启后已命中本地/内存缓存的图块不再进入跨服发布器、导致 MAP bundle 只能恢复一块的问题；缓存命中现在同样经过逐图块节流发布，城区会随实际浏览渐进恢复。

- 新增可选的 ks-Eco 跨服 MAP 窄桥：仅在 PNG 图块完成渲染并冻结为字节后发布，不在数据库工作线程读取 Bukkit `World`/`Chunk`。
- 新增 `cross-server.map-publish` 门禁、逐图块发布节流与本地载荷上限；默认关闭，ks-Eco 缺失或投影未启用时安全降级为纯本地地图。
- 同一世界/维度的多张图块现在组成有界 `ks-hwp-map-tile-bundle/v1` 快照；超过载荷上限时淘汰最旧图块，不再由后一张覆盖前一张。
- 新增 3 项窄桥合同测试，验证冻结数据、维度映射以及同源多图块共存；Leaves 实机最终读取 3 个图块、非 stale、节点在线。

## 2026-07-22

- 地图渲染把 ChunkSnapshot 分批抓取留在服务器线程，着色、合成、PNG 和磁盘缓存移到工作线程。
- zoom 2/4/8 不再递归生成大量中间图块；同图块并发请求复用 future。
- 地图标注 schema 改为 MariaDB 可用的 VARCHAR/元数据索引边界。
- 最终构建通过；Leaves 地产地图实机验收正常。
