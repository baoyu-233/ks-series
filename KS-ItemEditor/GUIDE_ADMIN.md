# ItemEditor 管理员使用指南

> [English](GUIDE_ADMIN.en.md) | 中文

> 你是服主/管理员，拥有 `itemedit.admin` 权限。本指南涵盖全部功能。

---

## 一、获取权限

OP 自动拥有 `itemedit.admin` 权限。如需单独授权：
```
/lp user <玩家名> permission set itemedit.admin true
```

---

## 二、GUI 物品编辑器

手持任意物品，输入 `/itemedit` 或 `/ie`。

### 修改名称
点击 **修改名称** → 在聊天框输入新名称 → Enter 确认。

支持格式：
- `&` 颜色码：`&6神剑` → 金色"神剑"
- MiniMessage：`<gold>神剑</gold>`
- 组合使用：`&6&l神剑` → 金色粗体"神剑"

### 修改 Lore
点击 **Lore** → 进入 Lore 编辑界面：
- 点击 **新增一行** → 聊天框输入
- 点击已有行 → 编辑
- 上下箭头 → 调整顺序

### 修改附魔
点击 **附魔** → 翻页浏览：
- **左键** +1 等级
- **右键** -1 等级  
- **Shift+左键** +5 等级
- **Shift+右键** 移除附魔

原版附魔可突破上限至 **32767** 级（在网页编辑器中更方便）。
FotiaEnchantment 附魔**不可突破**其配置的自然上限。

### ItemsAdder 套模型
前提：服务器已安装 ItemsAdder。

- 点击 **模型** → 浏览列表选择
- **左键** = 保留本体（仅复制 `item_model` + `custom_model_data`）
  - 推荐用于武器——保留伤害/耐久/可附魔，只改外观
- **Shift+左键** = 替换为 IA 物品（迁移名称/Lore/附魔）

### 模板操作（GUI 内）
- **📋 加载模板** — 输入模板码（含 `pl-`/`op-` 前缀），加载后可在 GUI 中继续编辑
- **📤 导出管理模板** — 生成 `op-xxxxxxxx` 格式的管理员专属模板
- **📤 导出玩家模板** — 生成 `pl-xxxxxxxx` 格式的玩家模板（已自动剥离 FE/IA/属性）

## 三、管理员网页编辑器

输入 `/itemedit web` 获取链接，在浏览器中打开。

### 相比 GUI 编辑器的增强功能
1. **物品预览面板** — 实时渲染名称+Lore 效果
2. **材质选择器** — 搜索 + 分类标签，选择任意物品材质
3. **FotiaEnchantment 附魔** — 分类筛选（近战/护甲/远程/工具/通用），冲突自动检测
4. **物品属性修饰符** — 27 种原版属性，自由组合
5. **ItemsAdder 模型** — 搜索+选择
6. **特殊符号面板** — 6 分类，点击插入
7. **选区格式化工具栏** — 选中文本一键包装格式码
8. **模板系统** — 可视化保存/导入/预览/修改/导出

### 模板码格式
- 你创建的**管理模板**：`op-xxxxxxxx`（玩家无法加载）
- 你创建的**玩家模板**：`pl-xxxxxxxx`（所有玩家可加载）

### 模板导入工作流
1. 你知道一个模板码（玩家或管理员均可）
2. 在"导入模板"框中输入，点击 **📥 导入预览**
3. 模板数据加载到编辑器，你可以：
   - 预览所有配置
   - 修改任意参数
   - 点 **💾 保存玩家模板** 或 **💾 保存管理模板** 导出为新模板
   - 原模板不受影响

## 四、FE 附魔兼容性说明

### FE 冲突检测
管理员模式下，网页编辑器会自动检测 FE 附魔内部冲突（如精准切割 vs 矿石爆破）。
冲突附魔旁显示 ⚠️ 图标，列表底部显示冲突详情。

### 配置文件
在 `plugins/ItemEditor/config.yml` 中可配置：
```yaml
fe-enchantments:
  admin-max-enchant-level: 32767  # 原版附魔上限（对 FE 附魔无效）
  ignore-vanilla-conflicts: false # 设为 true 允许互斥原版附魔共存
  ignore-fe-conflicts: false      # 设为 true 跳过 FE 冲突检测
  show-compatibility-warnings: true # 设为 false 隐藏所有冲突警告
```

### FE 附魔等级限制
FE 附魔**无法突破**其 YML 配置中的 `max-level`。即使管理员在网页编辑器中把输入框上限调到 32767，实际写入 PDC 时 FE 会强制限制到自然上限。这是 FE 插件的内部行为，ItemEditor 无法绕过。

达到自然上限时网页编辑器会显示金色「**已达上限**」徽章。

## 五、物品属性修饰符

### 可设置的属性（27 种）
| 类别 | 属性 |
|------|------|
| 战斗 | 攻击伤害、攻击速度、攻击击退 |
| 防御 | 护甲、盔甲韧性、击退抗性、爆炸击退抗性 |
| 生命 | 最大生命值、最大吸收、摔落伤害倍率、安全摔落距离、燃烧时间 |
| 移动 | 移动速度、飞行速度、跳跃强度、台阶高度、重力、水中移动效率 |
| 交互 | 方块交互范围、实体交互范围、方块破坏速度 |
| 其他 | 体型缩放、幸运、跟随范围 |
| 玩家 | 挖掘效率、潜行速度、水下挖掘速度、横扫伤害比率 |

### 运算方式
- **加法（直接加算）**：数值直接添加到基础值
- **基础倍率（基础值乘算）**：数值 × 物品基础属性值
- **总倍率（总值乘算）**：数值 × 当前总属性值（含其他修饰符）

### 生效插槽
- `any`：任意位置生效
- `mainhand` / `offhand`：主手/副手持有时生效
- `head` / `chest` / `legs` / `feet`：对应装备槽

## 六、Web 服务器配置

```yaml
web-server:
  enabled: true
  port: 8123
  bind-address: "127.0.0.1"  # 仅本机可访问（安全）
  token-timeout-seconds: 600 # 10 分钟，网页自动保活续期
```

**安全建议**：
- 默认 `127.0.0.1` 仅本机浏览器可访问
- 如需局域网访问改为 `0.0.0.0` 并设置防火墙
- 每个链接 10 分钟有效，过期需重新 `/itemedit web`

## 七、常见问题

**Q: 玩家能看到我的 FE 附魔吗？**
A: 不能。玩家创建的模板会自动剥离 FE 附魔、IA 模型和属性修饰符。你加载玩家模板时也看不到这些字段。

**Q: 旧模板（无 pl-/op- 前缀）还能用吗？**
A: v1.3.0 启动时会**自动清理**所有旧格式模板。升级前如有需要保留的模板请提前备份。

**Q: 属性修饰符不生效？**
A: 确认物品拿在正确的槽位（如 mainhand 属性需要拿在主手），确认运算方式正确。
