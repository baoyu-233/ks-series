## 你需要知道的范围

本指南只介绍实际面向玩家的玩法和命令：

- 保留市场、交易、银行、企业、税收、房地产、副本、政治、地图、称号、技能、物品继承和 KSBot。
- 不介绍 `ks-Sentinel` 管理员审计、`ks-BotGuard`、`ks-BossCombat`、`ks-Maintenance` 和 `ks-Compat` 的通用兼容功能。
- KSBot 是例外：虽然它由 `ks-Compat` 提供，但本指南保留玩家使用部分。
- 所有管理员命令，包括 `ks-Eco` 管理命令，都不放在本指南；完整后台命令请看技术总报告。

## 快速开始

```text
/eco gui                  打开统一经济 GUI
/market                   打开玩家市场
/balance                  查看余额
/kseco prices             查看官方收购价格、趋势和税率
/kseco web                获取玩家 Web 面板
/map                      获取世界地图
/title                    打开称号 GUI
```

如果某个入口提示“功能暂未开放”，说明服主通过经济功能门控暂时关闭了它，不是命令格式错误。

### 市场玩法

- 玩家可以出售带名称、Lore、附魔、PDC 和其他 NBT 的物品，购买者可以在预览界面查看完整物品信息。
- 官方收购只收购 `official-buy.default-items` 中的物品；当前动态价格由真实 SELL 流水的双向供需压力和均值回归漂移共同决定。
- 近期卖量超过历史基线会压低价格，近期卖量低于历史基线会提高价格；玩家买入不会直接推动官方收购价上涨。
- 市场保护价使用内部参考价、配方和物品 NBT 估值；它不是官方收购清单。
- 背包放不下的成交物品会进入 `/storage`，交易失败时会退款或恢复物品。

### 统一经济 GUI

`/eco gui` 或 `/kseco gui` 会打开统一入口，常用功能包括：

- 市场、官方收购、兑换、限时销售。
- 银行、企业、招投标、税收、政治。
- 房地产、副本、盲盒、主任务、补偿和交易物流。
- 玩家余额和资产概览。

具体模块没有安装或被关闭时，GUI 会显示离线或不可用状态。

### 盲盒

盲盒没有单独的玩家命令，进入 `/kseco gui` 后选择“盲盒”即可；服务器开启玩家 Web 面板时，也可以从 `/kseco web` 进入盲盒页面。玩家可以查看卡池价格、奖品预览、稀有度、个人保底进度和历史记录。

- 左键抽取 1 次，右键进行十连抽；实际按钮会根据当前卡池状态显示余额、资格和可用次数。
- 奖品按照稀有度和权重抽取，连续多次没有获得高等级奖品时，pity 保底会提高命中范围；抽到指定稀有度后重新计数。
- 奖励保留完整名称、Lore、附魔、属性、模型数据和 PDC。背包放不下时会自动进入 `/storage`，不会因为抽奖瞬间背包已满而消失。
- 企业盲盒需要企业权限，由企业公户按次支付；卡池可以要求最低企业等级，等级不足时会在扣款前拒绝。旧的共享预付票券已经退役。

玩家能看到的结果是：抽取成功会显示卡池、奖品和稀有度；十连会显示每次结果；保底命中会显示保底触发；背包满时会提示奖励已进入暂存箱。

### 限时商店

使用 `/limitedsale`（别名 `/lsale`、`/timesale`，需要 `kseco.limitedsale`）查看正在开放或即将开放的商品。每个商品会显示价格、剩余库存、开售和结束时间、个人限购以及自己已经购买的数量。

- 左键打开详情，查看直售商品的完整 NBT；盲盒商品会显示绑定奖池的预览。
- 右键购买 1 份，Shift+右键批量购买 10 份；支持整盒购买的商品可以用 Shift+左键购买整盒。
- 商品可能是固定物品，也可能是绑定盲盒池的限时盲盒。限时盲盒先完成商店扣款和库存检查，再从绑定池抽奖，不会重复扣两次盲盒价格。
- 库存不足、超过个人限购、尚未开售、已经结束或余额不足时，购买会被拒绝；购买过程中的失败会退款，背包放不下的商品进入 `/storage`。

例如一个总量 100、个人限购 3 的商品，购买一次后会显示剩余 99、个人已买 1；再次购买超过个人限购时不会先扣款。限时盲盒则会同时显示剩余可抽数量和每人最多可抽次数。

### 服务器补偿

补偿也在 `/kseco gui` 的“服务器补偿”入口中，不需要玩家执行管理员命令。这里显示服务器发布的维护补发、活动奖励或事故补偿，每份方案会标明每人数量和领取截止时间。

- 点击可领取的方案后，系统会记录你的领取状态，并把完整物品送入 `/storage`。
- 每个补偿方案每名玩家只能领取一次；已经领取的方案会显示已领取，重复点击不会重复发放。
- 方案过期或被停用后不能领取；如果补偿物品带有特殊名称、Lore、附魔、模型或 PDC，这些数据会完整保留。

## 二、经济系统设计思路

### 为什么同时有市场和官方收购

玩家市场解决“玩家之间如何自由定价”，官方收购解决“资源卖不出去时仍有最低流动性”。两者不是同一个价格表：官方收购只读取官方收购清单，市场保护价还会参考配方、内部材料价、附魔和潜影盒内容。

### 为什么物品要保留 NBT

服务器里的武器不只是一个 `DIAMOND_SWORD`。名称、Lore、附魔、属性、CustomModelData 和 PDC 都可能代表真实价值。因此市场、交易、兑换、盲盒、暂存箱和物品继承都优先保存完整物品数据，背包满时转入暂存箱，避免“钱结算了但物品丢失”。

### 为什么功能分成 GUI、Web 和命令

- GUI 适合玩家反复执行的操作，例如买卖、交易、兑换、银行和房产。
- 命令适合快速入口，例如 `/balance`、`/map`、`/dungeon start`。
- Web 适合查看长列表、图表、历史记录和跨模块管理。

因此同一个经济模块可能有三种入口，但底层使用同一套余额、物品和权限规则。

### 玩家能感受到的安全设计

- 背包满时进入暂存箱，而不是丢弃。
- 交易失败会退款或恢复库存。
- 房屋登记会先检查范围重叠和容积率，再一次性提交产权。
- 副本队伍有邀请时效、人数上下限和复活次数限制。
- 盲盒和补偿领取使用一次性记录，重复点击不会重复发放；盲盒的 pity 进度也会保存，不会因为重新打开 GUI 被重置。
- 限时商店同时检查库存、开售窗口和个人限购，购买失败会退款，不会出现“扣钱但没有商品”的正常路径。

### 盲盒、限时商店和补偿为什么分开

- **盲盒**把价值放在“概率、稀有度和保底”上，适合活动池和长期收集；玩家需要看到奖池规则，系统也要保存每次结果和 pity。
- **限时商店**把价值放在“明确价格、有限库存和时间窗口”上，适合限量材料、活动礼包和限时盲盒；它不会用随机概率替代商品本身的库存规则。
- **补偿**不是普通商品，也不参与市场供需。它是服务器对维护、活动或事故的可追踪发放，核心要求是有效期、完整 NBT 和每人一次。

### 其他系统的设计目的

- **银行不是第二个钱包**：存款、贷款和利率会改变货币流动速度，M0/M1/M2 用来观察全服资金是在玩家手里、活期账户里还是定期账户里。
- **企业不是聊天频道**：注册资本、公户、保证金、预付款和分红把企业资产与个人钱包分开，投标资格也因此有实际意义。
- **税收是经济调节器**：市场交易、产权转移、银行利息和企业利润可以使用不同税率，用来控制过热行业，而不是简单从每笔交易扣同一个比例。
- **土地和房屋是两层产权**：买地先获得使用权，登记房屋再获得可交易的建筑产权；容积率让城市规划成为真实限制，而不是只看有没有空地。
- **副本是高风险经济玩法**：门票、时间、队伍人数和递增复活费让“安全刷资源”和“高风险高收益”形成差异，副本房产则给长期队伍留下经营目标。
- **模块可以缺席但不能拖垮核心**：银行、企业、税收等 Extra 没加载时，玩家仍能使用市场和基本经济功能；对应 GUI 会显示不可用，而不是整套经济系统启动失败。

## 三、银行、企业、税收和政治

这些模块主要通过 `/eco gui`、`/politic` 和玩家 Web 面板使用。

### 银行

- 玩家可以查看银行列表、加入银行、存款、取款、申请贷款和查看还款状态。
- 银行利率受到央行基准利率和浮动范围限制。
- M0、M1、M2 货币供应量由系统后台追踪。

### 企业与招投标

- 注册私企或国企，使用企业公户参与采购、招标和分红。
- 企业拥有 1 级起的等级，由管理员按服务器运营规则设置；等级会影响部分企业卡池资格和企业地块福利倍率。
- 企业投标受到注册资本资格线约束，个人玩家也可以直接投标。
- 中标、预付款、保证金、采购和分红会记录到企业公户。

### 税收

- 市场交易、产权转移、银行利息、企业规模和分红可以适用不同税种。
- 企业税支持行业差异化和阶梯税率。
- 税率或提案状态受政治立法模式约束时，需要通过元老院流程变更。

### 政治

| 命令 | 别名 | 用途 |
|---|---|---|
| `/politic` | `/kspolitic`、`/senate` | 查看执政官、元老院、保民官、骑士和最新法案 |
| `/politic gui` | 同上 | 打开政治 GUI |
| `/politic appeal <提案ID>` | 同上 | 对合资格提案发起全服呼吁 |

政治面板和公告栏会显示选举、提案、投票、否决、覆议和已颁布法案。

## 四、房地产命令

### 地块

| 命令 | 别名 | 用途 |
|---|---|---|
| `/land` | `/myland`、`/myplot` | 打开我的地块和信任名单 GUI |

地块保护包括方块破坏/放置、容器、互动和爆炸。企业地块按企业成员授权，个人地块按所有者和信任名单授权。企业拥有的农业/工业地块福利会按企业等级倍率提高；玩家个人地块不套用企业等级倍率。

### 房屋

| 命令 | 别名 | 用途 |

| `/house wand` | `/myhouse wand` | 获取测量棒，左/右键选择两个角点 |
| `/house info` | `/myhouse info` | 查看当前选区 |
| `/house register <地块ID> <房屋名称>` | `/myhouse register` | 登记房屋并消耗住宅容积率 |

### 正在规划：生存 RPG

RPG 内容会逐步接入现有生存玩法，不会要求玩家转职或重练。机器产物、采集材料和市场货币会用于
制作基础装备；关键能力和装备突破还需要通过精英与副本获得不可交易的战斗证明。首个副本开放前，
会先推出少量区域精英、遗迹资源和协作目标。以上为规划内容，具体开放时间和规则以游戏内公告为准。

### 第一批 RPG 基础（测试服）

可用 `/ksrpg catalog` 查看材料兑换，使用 `/ksrpg exchange <兑换ID> [数量]` 完成兑换。第一批规则只有
单向材料精炼：8 个 Field Scrap 换 1 个 Refined Alloy，之后可依次精炼为 Conductive Coil、Stabilized Core
和 Relic Fragment。材料可交易；未来的战斗证明、关键能力与突破资格不可交易。

可用 `/rpggear` 或 `/gear` 打开装备栏。第一阶段只提供两枚戒指和一个护符槽，不会替代原版盔甲、背包
或副手。具体掉落、配方和开放范围以游戏内公告为准。

| `/inherit open` | `ksinherit.use` | 打开跨版本物品保存 GUI |
| `/inherit token` | `ksinherit.use` | 获取自己的 Web 状态链接 |

放入 GUI 的物品会完整保留名称、Lore、附魔、属性、模型和 PDC。管理员审阅和发放操作不在玩家版列出。

## 九、KSBot

KSBot 是玩家可以使用的 Leaves ServerBot 功能。每个 Bot 受名额、并发动作、动作间隔、重复次数和冷却限制约束。服务器默认限制每个 Bot 最多 4 个并发动作、每种动作最多一个，正数重复次数最多 100000；`-1` 表示有意持续执行，但仍受其他安全限制。

| 命令 | 权限 | 用途 |
|---|---|---|
| `/ksbot status` | `kscompat.bot.use` | 查看自己的名额和 Bot 状态 |
| `/ksbot rent <天数>` | `kscompat.bot.use` | 租用临时名额（启用经济模块时可用） |
| `/ksbot buy permanent` | `kscompat.bot.use` | 购买永久名额（启用经济模块时可用） |
| `/ksbot summon <名字> [皮肤名]` | `kscompat.bot.use` | 召唤自己的 Bot |
| `/ksbot tp <名字>` | `kscompat.bot.use` | 将自己的 Bot 传送到身边 |
| `/ksbot face <名字> [yaw] [pitch]` | `kscompat.bot.use` | 调整 Bot 朝向 |
| `/ksbot invincible <名字> [on\|off\|toggle]` | `kscompat.bot.use` | 切换 Bot 无敌状态 |
| `/ksbot nohunger <名字> [on\|off\|toggle]` | `kscompat.bot.use` | 切换 Bot 饥饿状态 |
| `/ksbot action <名字> list` | `kscompat.bot.use` | 查看 Leaves 支持的动作 |
| `/ksbot action <名字> start <动作> [间隔] [次数] [延迟]` | `kscompat.bot.use` | 执行动作 |
| `/ksbot action <名字> stop <序号\|all>` | `kscompat.bot.use` | 停止动作 |
| `/ksbot remove <名字>` | `kscompat.bot.use` | 移除自己的 Bot 并释放名额 |
| `/ksbotstorage open` | `kscompat.bot.storage` | 打开自己的 Bot 恢复物品箱 |
| `/ksbotstorage status` | `kscompat.bot.storage` | 查看恢复物品数量 |

Bot 因死亡、离线、服务器关闭或租期结束而回收时，系统会尝试把 Bot 背包物品放入恢复物品箱。领取后请确认背包有足够空间。

## 十、公告和 Web

| 命令/入口 | 用途 |
|---|---|
| `/announce` | 查看全服公告、法律和投票动态 |
| `/kseco web` | 玩家经济面板 |

Web 链接由服务器动态生成并带有短期 Token。不要把个人链接公开转发，也不要把管理员链接发给其他玩家。

## 十一、玩家版不包含的内容

以下内容仍存在于服务器或技术报告中，但不放进本玩家指南：

- `ks-Sentinel` 全服高危指令审计。
- `ks-BotGuard` 对 MythicLib/MMOCore 的 ServerBot 兼容保护。
- `ks-BossCombat` 的 Frostbound Boss 武器适配。
- `ks-Maintenance` 维护模式。
- `ks-Compat` 除 KSBot 外的 Fotia/Vulcan/外部插件兼容实现。
- `ks-core`、`ksHWP`、`ks-Title`、`ks-Skill`、`ks-Inherit` 的管理员配置命令。
