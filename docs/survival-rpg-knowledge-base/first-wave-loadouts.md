# 第一波战术配装规格

状态：内容规格，尚未写入 MMOItems 或副本奖励表。具体强度和技能以
`field-elite-and-stat-proposal.md` 为准。

## 目标

第一波装备服务于已进入中后期生存的玩家，建立在其满附魔下界合金甲、机器和经济积累之上。
它在第一座副本开放前提供明确但可自由组合的战斗倾向、武器技和护符技，并让玩家在进本时
为队伍承担不同责任。装备本身都可制作、交易或购买；决定后续突破的凭证仍只能来自战斗。

不采用固定四件套或职业锁定。当前 MMOInventory 的两个戒指和一个护符槽正好适合
模块化配装：

`一把主武器 + 两枚不同名称的戒指 + 一枚护符`

第一批不增加自定义护甲。生存甲装继续由原版与现有经济决定，避免把整套装备、强化、
随机词条和修理压力同时压到玩家身上。

## 副本前锁装

副本入口显示当前配置并由队长确认。每名成员只登记一把主武器、两枚不同 ID 的戒指和
一枚护符；实例创建后，这四个 RPG 装备位在本次副本结束前不能替换。原版护甲、食物、
药水和非战斗工具不受影响。

- 锁定的是本次副本内可生效的 RPG 加成，不扣押玩家物品。
- 不允许同一戒指 ID 同时放入两个戒指槽，防止单一最优词条叠两次。
- 一名玩家只可携带一个已登记的主武器进入战斗判定；备用武器不触发 RPG 被动、
  副本挑战记录或武器对策。
- 在城镇和野外可随时换装。只有进入实例时才锁定，因此不是永久职业选择。
- 副本 UI 应在确认页显示队伍的主武器、戒指倾向与护符倾向，但不强制队伍角色齐全。

这是一项未来 `ks-RPG` 的实例配装接口，不应由 MMOInventory 单独承担。MMOInventory
只负责存放两戒一护符；副本模板在服务器线程快照装备 ID，`ks-RPG` 保存该实例的不可变
配装记录。

## 装备质量与获取

| 层级 | 可用物品 | 获取方式 | 不包含的内容 |
| --- | --- | --- | --- |
| `KS_STANDARD` | 均衡武器、基础防御戒指 | 野外废料、原版材料、少量金币 | 战斗凭证、随机词条 |
| `KS_REFINED` | 专精武器、功能戒指、护符 | 精制合金、导能线圈、机器产线与玩家市场 | 副本通关要求 |
| `KS_RARE` | 有取舍的高端戒指与护符 | 稳固核心加区域精英掉落的可交易材料 | 首领凭证、遗物级突破 |
| `KS_RELIC` | 不在第一波直接发放 | 留给首领时代 | 成品毕业装备 |

每件制作物需要“生存材料 + 金币成本”。稀有件再加区域精英产出的可交易组件；它们可以
被市场供给，但不能绕过后续的凭证门槛。具体金币数字应在正式服读取前两周成交价后再填，
不在没有经济样本时硬定。

## 首批主武器

武器采用克制而不是纯数值升级。数值栏是初始测试目标，需以 MMOCore 的实际基础属性和
首领实测 TTK 校准后才写入物品 YAML。

| ID / 名称 | 类型 / 品质 | 初始属性目标 | 战术位置 | 代价 |
| --- | --- | --- | --- | --- |
| `SWORD:KS_TRAILBLAZER_BLADE` 开拓者长剑 | SWORD / `KS_STANDARD` | 中等攻击，略快攻速 | 最稳的持续输出；适合首次进本、补位与处理小怪 | 没有爆发或远程优势 |
| `SPEAR:KS_RELAY_PIKE` 接力长矛 | SPEAR / `KS_REFINED` | 中高攻击，小幅攻击距离或命中容错，轻微攻速惩罚 | 走位输出、处理需要保持距离的点名与救援窗口 | 被首领的长柄武器对策压制，必须等团队制造破绽 |
| `HAMMER:KS_QUARRY_MAUL` 采掘重锤 | HAMMER / `KS_REFINED` | 最高单次攻击，显著低攻速与低机动性 | 团队成功打断或解救后，吃短暂易伤窗口的爆发位 | 锤类减伤阶段伤害很低，错过窗口损失最大 |
| `BOW:KS_WINDWARD_BOW` 望风短弓 | BOW / `KS_REFINED` | 中等远程伤害，较快拉弓或投射物速度 | 白雾、地面危险和近战无法贴身时保持输出；负责观察与报点 | 近距离和高压救援效率较差 |

首领已经存在的锤、长矛 50% 减伤正好为第二、第三件武器创造了“等待团队完成机制后
再爆发”的共同目标。长剑和短弓不是更弱，而是稳定性选择，保障随机队也能完成学习局。

## 首批戒指

戒指分为生存、输出、机动三类。锁装验证限制每次最多一枚输出戒指，防止两个戒指都堆向
同一个伤害乘区；另一个槽必须在生存或机动中选择。现有三件基础饰品保留并纳入此表。

| ID / 名称 | 倾向 / 品质 | 初始属性目标 | 配装意义 |
| --- | --- | --- | --- |
| `RING:KS_RING_OF_BASTION` 堡垒之戒 | 生存 / `KS_REFINED` | 现有 +2 最大生命 | 给重锤、学习期近战和救援者的最低风险选择 |
| `RING:KS_RING_OF_RESOLVE` 决意之戒 | 生存 / `KS_RARE` | 现有 +4 最大生命，后续可小幅减伤 | 用资源换容错，不提高清场速度 |
| `RING:KS_RING_OF_MOMENTUM` 势能之戒 | 输出 / `KS_REFINED` | 小幅攻击速度 | 长剑连段、短弓持续输出与解救冰牢的命中次数 |
| `RING:KS_RING_OF_FOCUS` 聚焦之戒 | 输出 / `KS_RARE` | 小幅暴击率或暴击伤害，二选一实现 | 将收益集中到成功的破绽窗口，不提供常驻大额面板 |
| `RING:KS_RING_OF_STRIDE` 行旅之戒 | 机动 / `KS_REFINED` | 小幅移动速度或冲刺恢复 | 躲白雾、转场与赶到被冻结队友身边 |
| `RING:KS_RING_OF_RESONANCE` 共鸣之戒 | 机动 / `KS_RARE` | 小幅技能冷却缩减；主动技能上线前改为低额移速 | 为日后的团队主动技能预留成长位，第一波不制造空白词条 |

`聚焦之戒` 的暴击实现与 `共鸣之戒` 的冷却实现必须先验证当前 MMOItems/MMOCore 版本的
属性名称和事件归属。验证前，不能仅因 YAML 可加载就认定战斗中有效。

## 首批护符

护符决定一套配置最想解决的问题。所有护符均为单槽互斥，保证一个玩家不能同时拿到
极限伤害、极限生存和极限机动。

| ID / 名称 | 倾向 / 品质 | 初始属性目标 | 未来钩子，不在第一波强行实现 |
| --- | --- | --- | --- |
| `TALISMAN:KS_PIONEER_TALISMAN` 先驱护符 | 均衡 / `KS_REFINED` | 现有 +3 最大生命 | 新人进入内容的无风险默认项 |
| `TALISMAN:KS_ANCHOR_TALISMAN` 定锚护符 | 生存 / `KS_REFINED` | 最大生命或防御，小幅抗击退 | 被点名、冻结或救援中的稳定性；以后可接“救援完成”触发 |
| `TALISMAN:KS_SIGNAL_TALISMAN` 讯号护符 | 团队 / `KS_RARE` | 低额移速或攻击速度 | 以后接打断成功、冰牢击破后的短时团队增益；不由物品直接发技能 |
| `TALISMAN:KS_HUNTER_TALISMAN` 猎手护符 | 输出 / `KS_RARE` | 小幅武器伤害或远程伤害 | 以后接首领易伤窗口的额外收益，但不突破一枚输出戒指的限制 |

## 可直接给玩家展示的五种构筑

这些是推荐起点，不是职业，也不禁止其他组合。四把主武器、六枚戒指的合法双选和四个
护符会产生足够多的搭配；排除“双输出戒指”后仍有 `4 x 14 x 4 = 224` 种有效配置，
玩家可以针对首领、队友和个人操作习惯自行开发玩法。

| 构筑 | 组成 | 适合做什么 | 队伍中的价值 |
| --- | --- | --- | --- |
| 稳步开拓 | 开拓者长剑 + 堡垒之戒 + 势能之戒 + 先驱护符 | 首次开荒、杂兵、需要连续命中的机制 | 不依赖易伤窗口，队伍缺什么都可补 |
| 破绽处决 | 采掘重锤 + 决意之戒 + 聚焦之戒 + 猎手护符 | 打断或冰牢解除后的短爆发 | 把团队协作换成显著伤害回报 |
| 救援接力 | 接力长矛 + 堡垒之戒 + 行旅之戒 + 定锚护符 | 转点、解救被冻结队友、保持中距离输出 | 先到机制点，完成后仍能持续输出 |
| 雾中观察 | 望风短弓 + 势能之戒 + 行旅之戒 + 讯号护符 | 白雾和地面危险阶段，观察首领前摇 | 安全持续输出，并为将来的打断支援预留位置 |
| 机动补位 | 开拓者长剑或接力长矛 + 势能之戒 + 行旅之戒 + 讯号护符 | 熟练队的多机制轮转 | 在伤害、位移与未来团队支援之间取平衡 |

## 实装顺序与验收

1. 先以 MMOItems 原生静态属性实现四把武器、六戒、四护符，并为所有物品使用 `KS_*` 品质。
2. 在 `ks-RPG` 增加“副本配装快照”数据对象和验证 API；只在实例创建、离开和结算时读一次，
   不做循环扫描。
3. 由 `ks-Eco-RealEstateDungeon` 在开本时调用该 API，实例结束后删除快照。MMOInventory 不写
   副本状态。
4. 对四人队测试：重复戒指拒绝、只登记一把武器、死亡与掉线恢复、退出实例解锁、队伍成员
   配装展示，以及锤/矛在首领破绽前后的实际伤害差。
5. 主动技能事件仅在静态属性和锁装都稳定后加入。首批可接的事件为：命中、暴击、受到伤害、
   冰牢击破、打断成功、首领易伤开始/结束；所有 Bukkit 事件仍须在服务器线程处理。

这批内容没有遗物成品、随机词条、装备抽卡或可交易的首领凭证。它让生存经济提供准备，
让团队机制决定上限，并为第一座副本后的真正突破保留空间。
