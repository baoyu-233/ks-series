# Future Survival RPG Blueprint

> [中文](future-development.md) | English

This is a long-term direction, not a commitment to class names, item numbers, story characters, or private assets. Each phase advances only when player behaviour, server performance, and the economy support it.

## Permanent Boundaries

The persistent survival world remains primary: homes, machines, land, trade, community relationships, and existing progress retain value. RPG supplies exploration, combat, and cooperation reasons rather than a mandatory progression track.

- Ordinary survival never requires a class, dungeon clear, or gear score.
- Currency and materials fund preparation, crafting, supplies, and recovery; decisive abilities and breakthroughs are earned in combat.
- Group content adds options and efficiency without disqualifying solo players from normal play.
- Unless a season explicitly says otherwise, player identity, homes, and core achievements persist.
- A future class is an optional combat specialisation, never a replacement player profile.

## Delivery Direction

`Gaze` is the first story beat: a unique observatory pass opens a short cinematic in which an old station wakes, its world anchor breaks, and an unknown observer notices the player. It records story state only, grants no combat proof, and forces no class choice.

The sequence is: relic prelude with several viable weapon/accessory builds; reliable hot-reloadable cinematics; regional investigation with varied contracts; one roughly four-player cooperative dungeon with account-bound combat proof; optional combat specialisations; and finally seasonal horizontal content.

Classes are introduced only after builds and group play are proven. Each must solve a visible combat problem, require investigation and combat proof rather than purchase, allow bounded respecialisation, never improve machines/land/market power, and leave base-build players viable.

## Release Gates

Initial combat equipment should feel at least as capable as fully enchanted netherite in practical play. Differentiation comes from decisions, not unchecked stat escalation. For every reward, record whether it is tradeable, automatable, infinitely repeatable, or able to replace combat proof; restrict rewards that create inflation or bypass mastery.

Before releasing any scene, region, dungeon, or boss, define limits for concurrent performance, reward supply, first-failure readability, reloadable content maintenance, and cleanup on completion, failure, disconnect, and shutdown.

Near term: accept the `Gaze` scene, balance field elites/nodes/first-wave loadouts, review live usage and performance, validate combat proof and cleanup in the first cooperative dungeon, then name and build optional specialisations.
