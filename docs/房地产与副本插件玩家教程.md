# 房地产 & 副本系统 玩家教程

> [English](房地产与副本插件玩家教程.en.md) | 中文

所有房地产操作都在网页面板完成（服务器管理员会给你登录地址），打开后点击"🏞 我的地产"标签；副本操作通过游戏内指令 `/dungeon` 完成。

---

## 一、房地产系统

### 1.1 基本概念

- **区域（Zone）**：官方/管理员划定的一片地图区域，有规划类型（住宅/商业/工业/农业）、基础价格、年税率，以及"容积率"（区域内最多能登记多少块地，0 表示不限）。
- **地块（Plot）**：区域内被你买下的一小块地，记录在你名下，每年要交税。
- 地块和区域是两件事：你不是买"区域"，是在区域范围内划一块地买下来。

### 1.2 怎么买地

1. 打开网页面板 → "🏞 我的地产" → "浏览可购"子页。
2. 地图上：
   - **绿色填充**的区域 = 可以购买
   - **蓝色虚线框**的地块 = 已经被人买走了
   - 普通拖动鼠标 = 平移地图；滚轮 = 缩放
3. **按住 Shift 再拖拽鼠标**，在地图上框选你要的那块地（会出现黄色虚线框，左下角实时显示框选的世界坐标）。
4. 框选完成后，下方区域列表会出现"购入"按钮，点击它。
5. 弹出的确认窗口里坐标会自动填好（可以手动改），确认后系统会从你的钱包扣款（价格 = 该区域的基础价格），并把地块记到你名下。
   - 如果你是以企业身份购买，扣款会从企业公户（CORP-BANK 托管）里出。

> 注意：如果区域设置了容积率上限，且已经卖满了，即使你框选了空地也会购买失败 —— 提示你换一块地或找管理员加容积率。

### 1.3 我的地块 / 缴税

- "我的地产" → "我的地块"子页，能看到你名下所有地块：所属区域、世界、坐标范围、购入价、年税率、购入时间。
- 当前版本年税**不会自动扣款**，也**不会因欠税收回地块**——税款记录会生成账单，建议定期主动结清，避免日后系统升级后被追缴（具体以服务器公告为准）。

### 1.4 区域类型 vs 地块用途

- 区域的"规划类型"（住宅/商业/工业/农业）目前只是分类标签，不会限制你具体怎么用这块地。
- 真正决定地块用途的是"房产功能"字段，常规地块默认是"住宅（RESIDENTIAL）"，副本内的特殊房产可以是：
  - `DUNGEON_PORTAL` 副本传送门入口
  - `SAFEHOUSE` 安全屋（用于存放贵重物品）
  - `SHOP` 商店
  - `INDUSTRIAL` 工业设施

## 二、副本系统（指令：`/dungeon`）

副本是一个独立的"虚空世界"实例，进去之后跟主世界完全隔开，打完/超时后自动回收。

### 2.1 指令一览

| 指令 | 说明 |
|---|---|
| `/dungeon` | 打开副本大厅面板，查看可选副本模板、当前队伍状态 |
| `/dungeon invite <玩家名>` | （队长）邀请玩家加入队伍，邀请 **2 分钟**内有效 |
| `/dungeon accept` | 接受刚收到的组队邀请 |
| `/dungeon party` | 查看当前队伍成员列表 |
| `/dungeon start <模板ID>` | （队长）购票开本，正式进入副本 |
| `/dungeon leave` | 离开当前副本或退出队伍 |
| `/dungeon revive` | 副本内死亡后，付费就地复活 |

### 2.2 组队规则

- 队伍只存在于内存里，**服务器重启会丢失队伍状态**，请尽量速战速决。
- 一个人也能开本，系统会把你自己当成 1 人队伍。
- 只有队长能发邀请、能开本；普通队员只能 `accept`/`leave`/`party`。
- 每个副本模板都有人数下限/上限（常见配置是 1~4 人），**统计在线人数**——队伍里有人离线不算数。开本时人数不在范围内会被拒绝，提示你调整队伍。

### 2.3 开本流程

1. `/dungeon` 查看大厅，记下你要打的副本模板 ID。
2. 如果要组队：队长 `/dungeon invite 玩家名`，对方 `/dungeon accept`。
3. 队长确认人数达标后 `/dungeon start <模板ID>`。
4. 系统会从**队长**钱包扣门票费（常见默认 500，具体看模板设置），然后异步准备场地（清空一块虚空网格、贴入预制地图、扫描刷怪标记），完成后自动把全队传送进去。
5. 副本有时限（常见默认 60 分钟，模板可调），超时会被强制结束。

### 2.4 死亡与复活

- 副本内死亡不会直接传出副本，可以用 `/dungeon revive` 原地付费复活。
- 复活费用是**指数增长**的，公式：

  ```
  本次复活费 = 基础复活费 × 倍率^(已复活次数-1)
  ```

  按默认配置（基础费 200，倍率 1.8）：

  | 第几次复活 | 费用 |
  |---|---|
  | 第1次 | 200 |
  | 第2次 | 360 |
  | 第3次 | 648 |
  | 第4次 | 约 1165 |
  | 第5次 | 约 2097 |
  | ...第10次 | 约 84,717（达到上限，**第11次起无法再复活**） |

  默认每人最多复活 **10 次**，超过就只能 `/dungeon leave` 退本。这个公式提醒你：副本越打越死，复活越来越贵，建议谨慎一点而不是一直送。

### 2.5 副本内房产（高级玩法）

打副本不只是清怪，副本地图里也能像主世界一样置业：

1. 在副本内找到合适的地块坐标，通过网页面板的副本房产购买接口下单（扣你个人钱包）。
2. 购买后地块状态是"已购入但未开发"，需要再缴一笔**固定 5000 金币**的开发费才能正式启用（status 变为"已开发"）。
3. 副本内的房产契税率固定为 **15%**（比主世界地产更高，算是副本资源的稀缺税）。
4. 副本被销毁/回收时，该副本里所有挂靠的房产记录会**一并清空**——副本房产是"临时资产"，不是永久产权，请不要把它当长期仓库用，建议只用来做短期周转或者 `SAFEHOUSE` 应急存放。

### 2.6 给地图作者/管理员的提示（普通玩家可跳过）

- 副本地图是提前用 WorldEdit/FAWE 做好的 `.schem` 文件，放在 `plugins/ks-Eco/dungeon_schematics/` 下，文件名对应副本模板里配置的 `schematic` 字段。
- 地图里放告示牌可以标记刷怪点和出生点：
  - 告示牌第一行写 `[mm]`，第二行写怪物名（如 `SkeletonKing`）→ 该点刷出对应 MythicMobs 怪物，等级用模板默认等级；写成 `怪物名:等级`（如 `Zombie:15`）可以单独指定等级。
  - 告示牌第一行写 `[spawn]` → 标记玩家进本后的出生点；如果地图里没放这个牌子，系统会用网格内最高的方块顶部当出生点。
- 没装 MythicMobs 插件时，`[mm]` 标记会被跳过，不会报错卡死开本流程。
