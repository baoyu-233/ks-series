# Economy Knowledge Base

> [中文](README.md) | English

This is the durable design, implementation-boundary, and verification library for the KS-Series economy. It answers
why a feature exists, which module/table/entry point owns it, which thread may touch an object, and whether an old
design is accepted, being redesigned, or retired.

## Index

- [platform.en.md](platform.en.md): platform layers, data contracts, thread rules, permissions, and entries.
- [domains.en.md](domains.en.md): market, bank, enterprise, tax, property, and blind-box domains.
- [decisions.en.md](decisions.en.md): accepted and open product/technical decisions.
- [enterprise-blindbox-tickets.en.md](enterprise-blindbox-tickets.en.md): enterprise blind-box ticket audit.
- [claude-import.en.md](claude-import.en.md): imported historical notes that were rechecked against current facts.
- [verification-log.en.md](verification-log.en.md): dated builds, deployments, snapshots, and open validation.

Current source, schema, and read-only runtime evidence outrank this knowledge base. Do not turn proposals into
implemented features or static audits into live tests.
