# Economy Decisions

状态含义：`ACCEPTED` 已确认并应继续遵守；`PROPOSED` 有证据支持但仍需产品确认；`SUPERSEDED` 已被新事实替代。

## ECO-001 官方收购与保护参考价分离

- 状态：`ACCEPTED`
- 决策：`official-buy.default-items` 是唯一直接官方收购目录；内部参考价、配方、附魔和潜影盒估值只做市场保护底价。
- 原因：避免把所有可估值物品都变成系统无限收购入口。

## ECO-002 动态价格只使用真实供给压力

- 状态：`ACCEPTED`
- 决策：真实官方 SELL 量形成双向供给压力；测试流水永久排除；漂移均值回归并受上限约束。
- 原因：买入行为不应被误算成资源稀缺，测试预览不应改变正式价格。

## ECO-003 完整物品数据是跨域契约

- 状态：`ACCEPTED`
- 决策：市场、交易、盲盒、补偿、暂存箱和继承系统保存完整 `ItemStack`，并只在服务器线程解析。

## ECO-004 玩家 Web 的产品边界

- 状态：`ACCEPTED`
- 决策：玩家 Web 用于查看和管理长列表，不增加市场购买、挂单、求购、物流和暂存箱领取。当前桌面优先，不做非必要手机适配。

## ECO-005 Extra 失败隔离

- 状态：`ACCEPTED`
- 决策：Extra 成功启用后才公开；缺失或失败时核心经济继续运行，对应页面显示离线/空状态。

## ECO-006 盲盒、限时商店与补偿分域

- 状态：`ACCEPTED`
- 决策：三者分别负责概率保底、明确库存窗口和运营纠错，不复用彼此的库存或领取语义。

## ECO-007 企业和银行使用显式权限

- 状态：`ACCEPTED`
- 决策：角色模板与个人授权共同决定权限；服务器端必须再次校验。企业 `BLINDBOX_DRAW` 只代表单次抽取权，不代表批量资金管理权。
- 来源：Claude 历史记忆提供定位，已由当前 `EnterprisePermissionService` 和 Access Provider 代码复核。

## ECO-008 退役当前“企业购买式盲盒票券”

- 状态：`ACCEPTED / RETIRED`（2026-07-16）
- 决策：已删除购买、列表、使用、Web 路由、游戏 GUI 和新表创建入口；历史 `ks_bb_tickets` / `ks_bb_ticket_log` 表不主动删除，仅作为休眠遗留数据保留。
- 原因：当前票券没有折扣、分配对象或独立奖励来源，只把公户即时扣款改成提前扣款并附加过期风险；测试数据为零使用，并存在权限、门控、事务和线程问题。
- 详细证据：`enterprise-blindbox-tickets.md`。

## ECO-011 企业等级由核心统一管理

- 状态：`ACCEPTED`
- 决策：企业等级由 `EnterpriseLevelManager` 持久化并缓存，初版只允许管理员显式设置，不引入未经验证的经验或升级货币。
- 应用：企业盲盒池可设置最低等级；企业地块的农业/工业百分比福利按配置倍率缩放。玩家地块不受影响。
- 约束：资格、容量和治理解锁优先于永久伤害或无条件金币倍率。
- 候选扩展：采购/项目资格、并行订单数、企业仓库容量、银行授信和抵押率、成员上限、分红治理、城区地块资格、企业合同/副本层级。
