# Economy Domain Map

## 总体循环

```text
资源产出 -> 玩家市场定价 -> 官方托底/保护 -> 企业与玩家消费 -> 税收与费用回收
                     |                         |
                     +-> 银行提供流动性         +-> 地产、副本、盲盒、物流
```

经济系统的目标是让资源和货币形成可追踪的循环，而不是只增加金币入口。

## 领域职责

| 领域 | 主要实现 | 经济职责 | 关键不变量 |
|---|---|---|---|
| 玩家市场 | `ListingManager`、`MarketManager` | 玩家定价、SELL/BUY 挂单和成交 | 数量领取原子化；钱与物品失败时可补偿 |
| 官方市场 | `OfficialBuyManager`、`OfficialMarketSweepManager` | 提供最低流动性并收购低于保护价的挂单 | 官方收购清单与市场保护参考价严格分离 |
| 动态价格 | `PriceEngine`、`MarketValueService` | 用真实供给和受控漂移调整官方收购价 | 测试交易不污染真实供需；配方估值终止循环 |
| 物品存储 | `StorageManager` | 接收背包溢出和延迟领取物品 | 完整 NBT/PDC；合法堆叠；不静默丢失 |
| 玩家盲盒 | `BlindBoxManager` | 概率、稀有度、保底和长期收集 | 单抽/十连推进同一保底；记录结果；物品主线程解码 |
| 限时销售 | `LimitedSaleManager` | 明确价格、时间窗口、库存和个人限购 | 最新价格/库存事务复核；失败退款或回滚计数 |
| 补偿 | `CompensationManager` | 维护、事故和运营纠错发放 | 每人一次；有效期；领取与入库同一事务 |
| 银行 | Bank Extra | 存款、贷款、央行利率和 M0/M1/M2 | 银行权限独立；贷款发放与审批分权 |
| 企业 | Enterprise Extra + `EnterpriseLevelManager` | 法人、公户、等级、采购、招投标、工资和分红 | 企业资产与个人钱包分离；等级由核心统一读取；资金动作要求明确权限 |
| 税收/政治 | Tax/Politic Extra | 调节行业过热并治理税率变更 | 税种分域；立法模式下通过提案流程变更 |
| 房地产 | RealEstate Extra | 土地使用权、建筑产权和城市规划 | 地块与房屋两层产权；范围/容积率事务检查 |
| 副本 | Dungeon Extra | 以门票、时间和复活费交换高风险收益 | 实例生命周期清理；副本房产不是永久资产 |

## 已确认的边界

- 官方收购目录只来自 `official-buy.default-items`；内部参考价格只用于市场保护和估值兜底。
- 真实官方 SELL 流水进入供给压力；玩家 BUY 不直接抬高官方收购价。
- 盲盒解决随机与保底，限时商店解决窗口与库存，补偿解决运营纠错，三者不能合并成一个“发物品”模块。
- 银行不是第二个钱包；企业不是聊天频道；税收不是每笔交易统一扣点。
- 企业公户购买的奖励当前发给实际操作员工的背包/暂存箱，而不是企业仓库。这是需要明确治理规则的价值转移。

## 当前线程改造状态

- 已分离：官方保护估值/收购、玩家挂单领取、补偿领取、限时购买、玩家单抽/十连/限时盲盒批量。
- 待分离：求购单创建/履约/取消、企业盲盒，部分银行/企业/投标/邀请/地产/税务 GUI。购买式票券已退役，不再作为线程改造目标。
- 基础设施待办：有界计算池、串行数据库写入器与队列指标、持久化 settlement journal。
