# ks-Series Bug Hunt - 2026-07-18

> English | [中文](BUG-HUNT-2026-07-18.md)

Status: the first audit pass is complete. Twenty-five agents performed a read-only audit. Only findings with a concrete source
location and trigger sequence are recorded.
Follow-up maintenance has since landed code fixes in three batches without deployment: batch 1 covered dungeon revive/rewards,
enterprise dividend startup races, and bank/tax/Inherit settlement; batch 2 covered HWP auth/path, ItemEditor session/template,
Title image bounds, and RealEstate fail-closed protection; batch 3 covered InstanceWorld delayed-release occupancy guards and
cleanup bounds, LimitedSale conditional stock plus pinned cash backends, and RPG proof-gate / candidate-config reload hardening;
batch 4 covered irreversible-majority senate finalization, Sentinel failed-batch requeue, and KSBot infinite state-action blocking.
Those modules compiled or tested green. No JAR was deployed and Paper was not started. Cross-server transport/cursor/cache/lease,
MySQL/PostgreSQL business schema, currency idempotency/fee-split issues, BotGuard listener restore, Maven fake-green/JUnit
conversion, and ticket settlement journals remain open.

## Confirmed P0

- Dungeon revive accepts invalid instances and non-DEAD participants, can charge repeatedly, ignores a zero-row
  participant update, and still reports success.
- Dungeon completion commits `COMPLETED` before rewards. A crash or the first reward exception permanently drops
  some or all rewards because retry plans contain no reward recipients.
- Enterprise dividend recovery is not server/lease scoped. A second server can mark another server's in-flight
  payout as compensation-required, creating a later double-payment path.

## Confirmed P1

- Cross-server polling can skip late-committed older events, shares one cursor across servers, and never reloads a
  cursor after CAS failure.
- Transport idempotency compares records containing `byte[]` by reference, so an identical republish is rejected as
  conflicting content.
- Cache invalidation treats node HLC values as a global commit order; clock skew and listener failures can leave
  caches permanently stale.
- The ks-core connection layer supports remote databases, but current ks-Eco, bank, enterprise and politics schemas
  still contain SQLite-only DDL and DML and cannot run cleanly on MySQL/PostgreSQL.
- Limited sales can oversell across servers and may refund to a different balance backend than the original charge.
- Enterprise Vault calls can time out to the caller and execute later; partial dividend retry can repay successful
  recipients.
- Bank loans are committed ACTIVE before principal delivery, leaving unrecoverable missing payouts after a crash.
- Tax collection can lose the refund path when connection acquisition throws after Vault withdrawal.
- Real-estate cache refresh/reload failures can publish fail-open snapshots or preserve revoked trust indefinitely.
- HWP annotation search exposes private annotations without an owner/public predicate.
- HWP debug is not admin-gated and performs Bukkit inspection plus a blocking tile-render test for anonymous callers.
- ks-Eco has no JUnit/TestNG engine; the current main-based test classes result in zero Surefire tests and false-green
  `mvn test/package` runs.

Additional confirmed P1 findings include stale InstanceWorld releases freeing a reused grid, schematic cleanup outside
the configured arena, ItemEditor session overwrite and destructive template replacement, duplicate Inherit delivery,
RPG proof-gate type fail-open, invalid RPG YAML replacing the live catalog, early political vote finalization, HWP
recursive cache deletion, Title image decompression bombs, Sentinel audit loss, KSBot infinite-action bypass, MySQL
currency ID collisions, exchange fee splitting, stale lease resurrection, and BotGuard duplicate listener restoration.

The Chinese report contains source locations and reproduction details and will continue to receive the full findings.
